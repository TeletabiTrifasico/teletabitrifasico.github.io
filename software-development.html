Project 1: Chapeau

Chapeau is a desktop application I built as part of a team project to manage the full operations of a restaurant. It’s developed in C# using .NET 8.0 and Windows Forms, and it handles everything from taking orders and tracking stock to managing staff roles. One of the best things about the project is how each type of employee waiter, bar staff, kitchen, or manager gets their own custom interface and access level. It really helped us think through how different roles interact with the system in a real restaurant setting.

On the technical side, the app is structured using a layered architecture. We kept the logic clean by separating the code into different projects for models, data access (DAL), services, and UI. Everything is backed by a SQL Server database, which keeps track of orders, inventory, employees, and more. This project really helped me get comfortable with building larger-scale desktop apps, structuring code in a maintainable way, and thinking like a developer working on real business software.

Version 2

Building on my experience, I created a second version of the webshop with a more modern, professional stack. This version uses a PHP 8 backend connected to a Vue.js 3 frontend and a MySQL 8 database, all running inside Docker containers. The platform includes real-time shopping cart management, a full checkout process with order confirmation, user authentication through JWT tokens, and a role-based access system for users and admins. Admins can manage products, users, and orders via a dedicated dashboard, while users can browse products with detailed views and upload images for listings. Project 3: Garden Group

For my third project, I worked on a ticketing system for Garden Group, developed in C# and XAML as part of a university project. This application was designed to streamline issue tracking and support requests within an organization.

Project 4: Haarlem Festival

For my fourth project, I have been working on Haarlem Festival, the most complex and ambitious project I have tackled so far. This dynamic web application is designed to manage and showcase events and festivals in Haarlem, Netherlands, providing an intuitive platform for both visitors and administrators.

The application follows a modern tech stack, with a Vue.js 3 and TypeScript frontend that ensures a responsive, interactive user experience, and a Laravel 11 backend integrated with MySQL for efficient data management. Using Inertia.js, the system seamlessly connects the frontend and backend for a smooth, SPA-like navigation experience.

Key Features:

  • Festival Showcase: Displays upcoming festivals, event details, interactive schedules, and image galleries.
  • Ticketing System: Allows users to purchase tickets for events through an integrated interface.
  • Admin Dashboard: Enables CRUD operations for festivals, event scheduling, and user role management.
  • CMS System: Includes a rich-text editor, content versioning, and a hierarchical content structure.
  • Security & Authentication: Implements role-based access control, CSRF protection, and secure password hashing.

This project has significantly enhanced my expertise in modern JavaScript frameworks (Vue.js 3, TypeScript), Laravel backend development, database design, security implementation, and frontend-backend integration. It has been an incredible learning experience, allowing me to develop a scalable, feature-rich web application that meets real-world event management needs.

Project 5: Code Generation

For my fifth project, I collaborated in a team of four to develop a banking application consisting of a full REST API and a frontend interface. The backend was built using Java and the Spring Boot framework, with the API documented and accessible through Swagger UI. We started by designing a complete API specification with OpenAPI, which served as the foundation for development.

The application supports core banking functionalities and is secured with JWT-based authentication. It includes full unit testing (Java) and functional testing using Cucumber, aiming for high code quality and 90%+ test coverage. The frontend, built with Vue.js, connects seamlessly to the API and is fully responsive across devices. The entire system was deployed and hosted publicly using containerized environments.

This project strengthened my skills in backend development with Java, API-first design, frontend integration, and professional software development practices like testing, containerization, and CI/CD pipelines.
